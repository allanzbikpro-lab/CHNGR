🐙 GitHub Pages — Hébergement Gratuit

Héberger Chngr
sur GitHub Pages

GitHub Pages transforme ton dépôt de code en site web accessible depuis n'importe quel appareil. Gratuit, sécurisé en HTTPS, et ta Chngr aura une vraie URL permanente pour la synchro Raccourcis.

✓ 100% Gratuit 🔗 URL permanente 🔒 HTTPS automatique ⏱ ~15 minutes
🌍
Accessible partout

Ton app Chngr sera disponible depuis ton iPhone, iPad, Mac ou n'importe quel navigateur via une URL fixe.

⚡
Raccourcis iOS prêt

L'URL permanente GitHub Pages sera ta vraie URL de synchro à coller dans le raccourci Apple Santé.

🔄
Mise à jour facile

Pour mettre Chngr à jour, tu remplace juste le fichier sur GitHub. Le site se met à jour automatiquement.

1
Étape 1 sur 6

Créer un compte GitHub

Si tu as déjà un compte GitHub, passe directement à l'étape 2.

1
Aller sur github.com

Ouvre github.com dans ton navigateur. Clique sur le bouton « Sign up » en haut à droite.

🔒 github.com
🐙
Bienvenue sur GitHub
La plateforme pour les développeurs — et les non-développeurs aussi.
Sign up
Sign in
2
Remplir le formulaire d'inscription

GitHub va te demander :

Email — ton adresse e-mail principale
Mot de passe — minimum 8 caractères
Username — ton identifiant public (ex: thomas-martin). Ce nom fera partie de ton URL finale : thomas-martin.github.io/chngr
💡
Choisis bien ton username — il sera visible dans l'URL de ton application. Opte pour quelque chose de court et professionnel. Il n'est pas modifiable facilement après coup.
3
Vérifier ton email et finaliser

GitHub envoie un email de confirmation. Clique sur le lien dans l'email pour activer ton compte. Tu es maintenant connecté.

2
Étape 2 sur 6

Créer un repository

Un repository (ou "repo") est le dossier en ligne qui contiendra les fichiers de Chngr.

1
Créer un nouveau repository

Une fois connecté, clique sur le « + » en haut à droite → « New repository ». Ou va directement sur github.com/new.

2
Configurer le repository

Remplis le formulaire de création :

🔒github.com/new
Repository name *
chngr ✓ Disponible
Visibilité
🔓 Public
Tout le monde peut voir ce repository. Requis pour GitHub Pages gratuit.
🔒 Private
GitHub Pages nécessite un abonnement payant.
✓
Initialiser avec un README
Create repository
⚠️
Repository en Public obligatoire pour utiliser GitHub Pages gratuitement. Tes fichiers HTML seront visibles (code source), mais tes données de pesées restent dans ton iPhone (localStorage) — elles ne sont pas dans les fichiers.
3
Étape 3 sur 6

Uploader les fichiers Chngr

On va déposer tes fichiers directement depuis le navigateur, sans ligne de commande.

1
Ouvrir ton repository sur GitHub

Tu arrives sur la page de ton nouveau repository qui ressemble à ça :

📁 ton-username / chngr Public

2
Uploader les fichiers via l'interface web

Clique sur « Add file » → « Upload files ». Une zone de glisser-déposer apparaît.

Dépose les fichiers suivants (extraits du ZIP Chngr) :

🔒github.com/ton-username/chngr/upload

Important : Le fichier principal doit impérativement s'appeler index.html (en minuscules). C'est ce fichier que GitHub Pages charge par défaut quand quelqu'un visite ton URL.
3
Valider le commit

En bas de la page, tu vois la section « Commit changes ». Laisse le message par défaut ou écris Ajout Chngr v1. Clique sur « Commit changes » (bouton vert).

✓
Tes fichiers sont en ligne ! Tu devrais voir index.html, guide-raccourcis.html et guide-github.html listés dans ton repository.
4
Étape 4 sur 6

Activer GitHub Pages

C'est l'étape qui transforme ton dépôt de code en site web accessible publiquement.

1
Aller dans les Settings du repository

Depuis la page de ton repository, clique sur l'onglet « Settings » (la roue dentée ⚙️ tout à droite des onglets).

2
Trouver la section « Pages » dans le menu gauche

Dans la colonne de gauche, fais défiler jusqu'à la section « Code and automation » et clique sur « Pages ».

3
Configurer la source de déploiement

Tu vois la page GitHub Pages. Configure-la ainsi :

🔒github.com/ton-username/chngr/settings/pages
GitHub Pages
GitHub Pages is designed to host your personal, organization, or project pages.
Build and deployment
Source
Deploy from a branch ▾
Branch
🌿 main ▾
/ (root) ▾
Save
→ Choisir la branche main et le dossier / (root)
Source : sélectionne Deploy from a branch
Branch : sélectionne main
Dossier : laisse / (root)
Clique sur « Save »
4
Attendre le déploiement (~2 minutes)

GitHub prépare ton site. Tu peux suivre l'avancement dans Actions (onglet en haut du repository). Une fois terminé, un point vert ● apparaît.

🎉
● Active
Ton site est en ligne à :
https://ton-username.github.io/chngr
ℹ️
Si la page Settings/Pages ne montre pas encore l'URL, actualise la page après 1-2 minutes. Le premier déploiement peut prendre jusqu'à 5 minutes.
5
Étape 5 sur 6

Accéder à ton application

Ton URL Chngr suit un schéma précis basé sur ton username GitHub.

Ton URL Chngr permanente
https://ton-username.github.io/chngr
1
Ouvrir l'URL dans Safari sur iPhone

Ouvre Safari sur ton iPhone et tape ton URL. Chngr se charge exactement comme si c'était une app native.

2
Ajouter à l'écran d'accueil (recommandé)

Pour une expérience vraiment app-like sur iPhone :

Dans Safari, appuie sur l'icône Partager (carré avec flèche vers le haut)
Fais défiler → « Sur l'écran d'accueil »
Nomme-le Chngr → « Ajouter »
📱
Chngr apparaît sur ton écran d'accueil comme une vraie app. Elle s'ouvre en plein écran, sans la barre d'adresse Safari, grâce aux balises apple-mobile-web-app-capable déjà dans le code.
3
Vérifier que les données persistent

Ajoute une pesée de test, ferme l'app, rouvre-la. Tes données sont bien là — elles sont stockées dans le localStorage de Safari sur ton iPhone, indépendamment du serveur GitHub.

💡
Astuce : Si tu vides "Historique et données de sites web" dans les réglages Safari, les données localStorage sont effacées. Pour sauvegarder, utilise la fonction d'export de Chngr régulièrement.
6
Étape 6 sur 6

Mettre à jour l'application

Quand une nouvelle version de Chngr est disponible, voici comment la déployer en 2 minutes.

1
Aller sur ton repository GitHub

Va sur github.com/ton-username/chngr. Tu vois la liste de tes fichiers.

2
Remplacer le fichier index.html

Clique sur index.html dans la liste → puis sur l'icône ✏️ crayon en haut à droite du fichier. Une autre option : clique directement sur « Add file » → « Upload files » et dépose le nouveau index.html — GitHub écrase automatiquement l'ancien.

ℹ️
Le déploiement est automatique après chaque commit. Attends 1-2 minutes puis recharge Chngr sur ton iPhone. Tes données de pesées ne sont pas affectées — elles vivent dans le localStorage, pas dans le code.
!
Forcer le rechargement sur iPhone

Safari met parfois le site en cache. Si tu vois l'ancienne version après une mise à jour :

Appui long sur le bouton Recharger ⟳ dans Safari
Sélectionne « Recharger sans le cache »
⚡
Étape bonus

Mettre à jour le Raccourci iOS

Maintenant que tu as une URL permanente, mets à jour la synchro Apple Santé.

⚠️
Action requise : Si tu avais configuré le raccourci avec un fichier local (ouvert depuis les Fichiers sur iPhone), tu dois maintenant remplacer l'URL par ta nouvelle URL GitHub Pages — sinon la synchro ne fonctionnera plus si tu changes de téléphone ou de navigateur.
1
Générer un nouveau token depuis l'URL GitHub Pages

Ouvre Chngr via https://ton-username.github.io/chngr dans Safari. Va dans le panneau Synchro Apple Santé → génère un token → copie l'URL complète. Cette URL contient désormais ton adresse GitHub Pages.

2
Modifier l'action « Ouvrir des URL » dans Raccourcis

Ouvre l'app Raccourcis → ton raccourci « Chngr — Sync Poids » → appuie sur les 3 points ··· pour modifier. Trouve l'action « Ouvrir des URL » et remplace l'ancienne URL par la nouvelle (qui commence par https://ton-username.github.io/chngr).

✓
Tester la synchro

Exécute le raccourci manuellement. Chngr s'ouvre depuis GitHub Pages et confirme la réception. L'automatisation matinale utilisera désormais cette URL permanente.
